# Sp1Benchmarks

apc-optimizer benchmark inputs generated from SP1 guests, mirroring the layout of
`OpenVmBenchmarks/`. Drop this folder into the apc-optimizer repo and drive it
with `OpenVmBenchmarks/benchmark.py` (point it at this directory).

## Sets

- `rsp/` — the top-ranked basic blocks of the rsp (reth) guest verifying eth
  mainnet block 21740137, ranked by cell PGO. The SP1 analog of `openvm-eth`.
- `keccak/` — the costliest software-keccak block(s) of the `keccak-bench` guest.

## Per-case files

- `apc_<rank>_pc<pc>.json.gz` — the unoptimized APC in powdr's `SymbolicMachine`
  format (flattened `{block, machine, subs, optimistic_constraints, bus_map}`).
  This is the apc-optimizer's input.
- `apc_<rank>_pc<pc>.powdr_opt.json.gz` — powdr's optimized machine
  (`{machine, bus_map}`), used as the reference the apc-optimizer is compared
  against. Snapshotted *after* `optimize()` but *before* `add_guards`, so it
  carries no `is_valid` guard column.
- `manifest.json` — ranking + per-case size stats.
- `apc_candidates.json` — powdr's full candidate dump (assembly view for the
  HTML report).

## Notes

- Field: SP1 uses **KoalaBear**; field elements are encoded accordingly (the
  apc-optimizer's parser is currently BabyBear — that's the consuming side to
  reconcile, not these inputs).
- Ranking is cell PGO's value ranking, which folds in powdr's effectiveness, so
  the top-N is close to but not exactly the N costliest blocks.

## Regeneration

Generated by `scripts/gen_sp1_benchmarks.sh` in the SP1 repo.
